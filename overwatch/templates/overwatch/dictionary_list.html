üìö How Does the Dictionary Work?

üéØ Overview

The Dictionary translates technical names into human-friendly names. When a server is created or updated, it automatically translates specific fields using the rules you define here.

‚öôÔ∏è How It Works

1. Agent Collects Data: The agent gathers raw hardware information (e.g., "HP", "Intel Core i5-7500T")

2. Submits to Server: Raw data is sent to the API

3. Auto-Translate: When saved, the Server model checks the Dictionary and replaces technical names with friendly names

4. Stored in Database: Translated values are saved (e.g., "Hewlett-Packard", "Intel Core i5 7th Gen")

üîÑ Supported Fields

  • Manufacture: "HP" ‚Üí "Hewlett-Packard"
  • Product Name: "HP Elite Slice G2 MS SRS Audio Ready" ‚Üí "HP Elite Slice G2"
  • CPU: "Intel Core i5-7500T" ‚Üí "Intel Core i5 7th Gen (Kaby Lake)"
  • OS: "Ubuntu" ‚Üí "Ubuntu Linux"

‚ûï How to Add an Entry

1. Click "Add entry" button above

2. Fill in the form:

  • Translate From: Which field to match (e.g., "manufacture", "product_name")
  • Original Keyword: The technical name to find (e.g., "HP")
  • Category: The type (Vendor, Platform, CPU, etc.)
  • Standardized Value: The friendly name to replace with (e.g., "Hewlett-Packard")
  • Is Active: Enable/disable this translation

3. Save and it will apply to new/updated servers

üîÑ Apply to Existing Servers

New translations automatically apply to:

  • Newly created servers (from agent)
  • Servers edited via the web interface

For existing data: Run the management command to update all servers:

python3 manage.py apply_translations

Add --dry-run to preview changes first
